What counts as “logs”?

“No logs” means the provider doesn’t store data that can tie your activity to you. The grey area is definitions:

  • Activity logs — sites visited, apps used, traffic contents. These should never be kept.
  • Connection logs — timestamps, session duration, incoming IP, server ID. Some keep limited metrics for operations; stricter no-logs avoid or anonymise them.
  • Account data — email/payment. Needed for billing; not a “VPN log” but still personal data.

If a provider says “no logs” but also mentions “abuse control via IP/time logs”, they’re keeping some connection metadata. Decide if that’s acceptable.

Marketing claims vs reality

Any VPN can print “zero logs” on a landing page. What matters is independent verification and an architecture that minimises data by design.

  • Audits & assessments: External firms review infrastructure and policies.
  • RAM-only servers: Reboots wipe state; configs boot from read-only images.
  • Private DNS: DNS resolvers run inside the tunnel; no clear-text queries to third parties.
  • Narrow telemetry: If used, it’s aggregate and can be disabled.

How to verify “no-logs” as a user

  1. Read the audit. Look for scope (apps + servers), frequency, and whether issues were fixed.
  2. Check transparency reports. Do they publish legal request stats and actions taken?
  3. Test DNS. Connect → run an Extended test (see below). You shouldn’t see your ISP’s resolvers.
  4. Inspect the apps. Kill switch, private DNS, protocol choice (WireGuard/OpenVPN) — these reduce leak risks.
  5. Review the track record. Past incidents, court docs, or third-party write-ups carry more weight than ads.

RAM-only servers & architecture

RAM-only (diskless) stacks reduce the chance of persistent data. Configs are baked into an image; on reboot, everything resets. Combined with export-controlled admin access, limited telemetry, private DNS and peer reviews, this creates defence in depth.

UK legality & requests

Using a VPN is legal in the UK. Providers may receive lawful requests about their infrastructure. A real no-logs provider can comply with a valid process while still having nothing useful to hand over about user activity. That’s the point of the model.

Leak tests (IP/DNS/WebRTC)

“No logs” is moot if you leak. Always test:

  1. Connect to a UK server (London/Manchester).
  2. Open a leak test site (e.g., dnsleaktest.com) → run Extended test.
  3. If you see your ISP’s DNS or your real IP, reconnect, switch protocol (WireGuard → OpenVPN TCP/443), enable leak protection, or try another server.
