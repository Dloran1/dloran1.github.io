Quick setup on macOS

  1. Install the provider’s macOS app (NordVPN, Surfshark) and sign in.
  2. Set Protocol: WireGuard/NordLynx. Enable Auto-connect on unknown Wi-Fi.
  3. Turn on Kill switch and DNS leak protection (in-app toggles).
  4. Connect to a nearby UK server (London, Manchester, Birmingham).
  5. Optional: exclude sensitive work/banking apps with split tunneling if needed.

On restrictive Wi-Fi (office/hotel), switch to OpenVPN TCP/443. More in the protocols guide.

Best settings for UK networks

  • Protocol: WireGuard/NordLynx first; TCP/443 only when UDP is throttled/blocked.
  • DNS: Use provider DNS inside the tunnel; avoid system/public DNS while connected.
  • Auto-connect: On for unknown Wi-Fi; optional on boot.
  • Split tunneling: Keep streaming inside the VPN; exclude apps that refuse VPN.
  • Updates: Keep the VPN app current — providers rotate IPs and fix detection.

Kill switch on macOS explained

On macOS, most VPN apps use a system extension and a per-app/network firewall to block traffic if the tunnel drops. It’s not the same as macOS’s built-in VPN profiles — use the provider app to ensure full blocking.

  • Test once: start a download → force-quit the VPN app → traffic should stop until reconnection.
  • If you use manual IKEv2 profiles, ensure “Send all traffic over VPN” is enforced (apps often do this automatically).

IP/DNS/WebRTC leak checks

  1. Disconnect VPN → check baseline IP (country/ISP).
  2. Connect VPN (UK) → confirm IP shows UK and a VPN ASN.
  3. Run extended DNS + WebRTC tests → ensure no ISP resolvers and no real IP exposure.

Flush DNS if needed: sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder

Streaming on Mac (Netflix UK, BBC iPlayer)

  1. Pick low-load UK servers. If you see proxy/unblocker messages, rotate UK city.
  2. Relaunch apps/browsers after server/protocol changes (they cache endpoints).
  3. On strict Wi-Fi, try OpenVPN TCP/443 and relaunch the app/site.

Fix slow speeds

  • Switch to a nearer or less-loaded UK server.
  • Use WireGuard/NordLynx first; if packet loss spikes, test TCP/443.
  • Close bandwidth-heavy background apps (iCloud/Photos sync, cloud drives).
  • Prefer Ethernet for 4K streaming or large downloads; Wi-Fi jitter causes buffers.
  • Reboot router/Mac after many changes to clear stale routes.

FAQ

Which protocol is best on macOS?

WireGuard/NordLynx for speed and stability; OpenVPN TCP/443 on restrictive networks.

Does macOS have a real kill switch with VPNs?

Provider apps implement blocking via system extensions/firewall rules. Use the app’s kill switch and test once.

How do I stop DNS leaks on Mac?

Enable DNS leak protection, reconnect the VPN, flush DNS (sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder), then re-test.

Do I need to disable iCloud Private Relay?

While testing or troubleshooting — yes. It can interfere with VPN DNS routing. Re-enable afterwards if you prefer.
