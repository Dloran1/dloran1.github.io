Quick 2-minute preset

  1. Protocol: WireGuard/NordLynx.
  2. Kill Switch: On (system-wide if available).
  3. DNS: Use provider DNS.
  4. Auto-connect: On login + on untrusted Wi-Fi.
  5. Server: Low-load UK city near you.

New to VPNs? Start with What is a VPN? and VPN Protocols.

Protocol choice

  • WireGuard/NordLynx — fastest; best efficiency.
  • OpenVPN UDP — fallback when WG is capped.
  • OpenVPN TCP/443 — for strict hotel/office Wi-Fi.
  • IKEv2 — stable on iOS while roaming Wi-Fi/LTE.

Security & privacy

App toggles

  • Kill Switch: block traffic outside the tunnel.
  • DNS: force provider DNS to avoid IP↔DNS mismatches.
  • Obfuscation: off by default; on only if blocked.
  • Auto-connect: on untrusted Wi-Fi; optional at login.

System tips

  • Firewall: keep enabled; allow the VPN helper.
  • IPv6: disable if not tunneled; or use a provider with IPv6.
  • Browser: mitigate WebRTC IP disclosure.

Streaming presets

Speed tips

  • Use WireGuard/NordLynx and low-load UK endpoints.
  • Prefer 5 GHz/Ethernet for 4K streams.
  • Pause heavy cloud sync (OneDrive/iCloud) during streaming.

Leak tests

  1. Verify your public IP = VPN country.
  2. Ensure DNS resolvers belong to the VPN.
  3. Mitigate WebRTC leaks in the browser.
  4. Reconnect & flush DNS if results look off.

FAQ

Which protocol is fastest in the UK?

WireGuard/NordLynx. On strict Wi-Fi use OpenVPN TCP/443.

Do I need a kill switch?

Yes. Enable the app’s kill switch so traffic is blocked if the tunnel drops.

How do I avoid streaming errors?

Keep IP and DNS consistent: use provider DNS, rotate servers when flagged, quit and relaunch the app after switching.

What if speeds drop in the evening?

Switch to a low-load endpoint, move to 5 GHz/Ethernet, and pause cloud syncs during streaming.
